Echo server

Exercise overview

In this exercise you will create a Node.js server, which reads the body of the incoming requests, and sends that same body as response. So, the server echoes the request.

Implementing the exercise

Start by making sure you are in your local clone of the Git repository, and in its exercises/02_http/2.4-echo-server directory where this instructions.html file is placed. Then:

  • In this directory run the command npm init to initialize your Node project. Use the default values offered by npm by pressing Enter.
  • Write your server code in index.js file you create inside this directory
  • Inside createServer method's callback, you need to capture the events that are emitted when message body is being received. For a quick reminder on how to do this, read Node tutorial's example
  • Use response's writeHead-method to set the correct HTTP response status code (200) and content type (text/plain) for the response.
  • Use response's write method to add the body to the response
  • Use response's end method to end adding to the response, and send it to the client
  • You can run your server by running the command node index.js in this directory
  • You can test the server for example with cURL. Something like the following should work: 
                curl --location --request POST 'localhost:3000' \
            --header 'Content-Type: text/plain' \
            --data-raw 'All together now!'
    . In the server's response the body should be 'All together now!'.
  • Correct all possible mistakes
  • Commit your index.js and package.json files, and push them to your Git repository.
