Routing

Exercise overview

In this exercise you get to implement a Node.js server, which is able to route incoming requests based on the path the request is directed to.

For example in request to localhost:3000/resources/23 the /resources/23 forms the path part of the URL.

Implementing the exercise

Start by making sure you are in your local clone of the Git repository, and in its exercises/02_http/2.6-routing directory where this instructions.html file is placed. Then:

  • In this directory run the command npm init to initialize your Node project. Use the default values offered by npm by pressing Enter.
  • Write your server code in index.js file you create inside this directory
  • When the server receives a request, you must read the path part of the URL. For example in request to localhost:3000/resources/23 the /resources/23 forms the path part of the URL. Read the Node tutorial for more information on how to do this.
  • Reply to request directed to path /classical by sending the contents of file homer.html from this directory
  • Reply to request directed to path /modern by sending the contents of file bradbury.html from this directory
  • For requests to the default path / respond with the contents of file index.html from this directory
  • For requests to any other paths respond with response that has HTTP status code 404 and statusMessage 'Requested content not found'
  • Use response's write method to add the body to the response
  • Use response's end method to end adding to the response, and send it to the client
  • You can run your server by running the command node index.js in this directory
  • Correct all possible mistakes
  • Commit your index.js and package.json files, and push them to your Git repository.
