Responsiveness implies the ability to adapt to a new circumstances, such as the view resized or being viewed with different size displays. In the exercise previous exercise, we implemented the CSS for the shop UI. This exercise extends the style declarations by making them responsive to the available screen size.

In this exercise you make the online shop page adapt to different device screen sizes using CSS grids. NOTE: media-queries must not be used. The columns in the grid must:

  • be at least 400px wide
  • if there is excess space, maximize the space usage, and occupy it all
  • in case whent there is not enough space available, be shown in one-column mode

Responsiveness must be handled with the following elements, which are to be placed side-by-side if space allows:

  • section
  • aside, and
  • nav
where sections and aside are inside main,
and navs in the footer.

Note: the solution does not have to be based on grid-area-templates.

The procedure is the following:

  • Review index.html and styles.css files in this directory
  • Add responsiveness to styles.css with grids, the appropriate places are marked with TODOs Check Plussa instructions.
  • Commit your changes to git repo
  • Submit your work to Plussa
