Media queries

Instructions

Responsiveness implies the ability of Web content to adapt to a new circumstances, such as the browser window being resized, or being viewed with on different sized displays. In a previous exercise we implemented the CSS for the UI shop. This exercise extends the layout by making it responsive, but now with media queries.

Make online shop adapt to different device sizes using CSS media queries. The sizes needed for this exercise are listed here:

Small phone < 600px
Phone 600px - 767px
Tablet 768px - 991px
Desktop 992px - 1199px
Wide desktop >= 1200px

Grids can be used as an assisting tool.

The procedure is the following:

  • Copy index.html and styles.css from ex4 to ex6
  • Add responsiveness to styles.css with media queries, like in the previous exercise with fluid grids. The layouts for the different screen sizes are as follows (the elements IDs remain the same as in previous exercises):
    • if size is small phone, section, aside, and nav elements reside in one column; the order is s1, a1, s2, a2, s3, n1, n2
    • if the size is phone or tablet, elements are set in two columns, the order remains the same, however, nav elements must go to a separate row
    • if size >tablet, elements are set in three columns. Now, the order changes a bit: first row consists of s1, s2, and a1, next row of s3 and a2; the last row of n1 and n2. Three columns in each row.
    • if there is need for empty elements, these are put in the middle.
    • Save your style definitions in styles.css.
  • Commit your changes to git repo
  • Submit your work to Plussa
