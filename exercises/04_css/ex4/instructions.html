CSS Grid layout

In this exercise the HTML elements in the page are positioned with CSS. See Fig.1 for the positioning of page elements that needs to be achieved.

  • Header is the first container element, a nav element without ID is nested with it.
  • Next, comes the main element with an article element in it. The article functions just as a container, it itself is not visible.
  • Inside the article, there are three section elements, and two aside elements.
  • Footer is the bottommost element under main.

The positioning and container-containee relationships must be implemented as described. In case of many containees, ids are necessary. Ids are unique, and they make it easier to point to a specific element e.g. when positioning them. Ids should be as described in the table:

Parent Tag Id
header nav -
article section sec1, sec2 and sec3
article aside aside1, aside2
footer nav nav1, nav2

CSS grid settings needed (example)

#container{
      display: grid;
      font-size: 2em;    
      grid-template-areas: "";
    }
    #a{grid-area:a}   // use ids defined below
    ...

The CSS layout of the online shop

You are going to build a general online shop UI with HTML. The shop uses such semantic tags as header, footer, nav, main, and inside main, multiple articles and sections, see table 1.

Table 1. tags and ids in online shop
Parent Tag Id
article section sec1, sec2 and sec3
article aside aside1, aside2
footer nav nav1, nav2

The grader checks the relative positioning: header must be the topmost element, followed by main and footer.

In main, it is important that sections are in the same column on the left, as well as aside elements on their own column on the right. In the footer, the two nav elements must be side-by-side.
