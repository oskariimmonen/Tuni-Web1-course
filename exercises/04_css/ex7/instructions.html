Accessibility

AXE - Browser Extension for Accessibility

During the web development accessibility plugins are handy in auditing the accessibility of pages. In this exercise a special Axe, 'accessibility auditing tool' is used. It is available as a browser extension for Chrome, Edge, and Firefox.

The extension can be found in its own tab inside the browser developer tools. (Developer tools can be opened with Ctrl + Shift + I). In Firefox, you can also use the built-in Accessibility tools that can be found on their own tab inside the developer tools.

Accessibility auditing is going to be integrated in some of the future exercises too.

Instructions

In the git repository, in the directory exercises/04_css/ex7, there is a file index.html. It shows a bad example of accessibility and your task is to fix it according to the accessibility rules, e.g., by putting ARIA labels and other labels in place.

Audit the accessibility with Axe, and fix all critical or serious issues that the tool reports. After fixing, check the accessibility once again with the developer tools before submitting the exercise. Submit to Plussa.
