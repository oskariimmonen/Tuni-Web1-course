3.3 Method Based Routing

Exercise Overview

In this exercise you create a Node server which will respond to any http GET request with one HTML file and to any POST request with another HTML file.

The server must have the following functionality:

  • Server listens on localhost port 3000
  • On every http GET request the server responds with the contents of get.html file in this directory.
  • On every http POST request the server responds with the contents of post.html file in this directory.
  • On every other type of request the server responds with a status code 405 (Method Not Allowed).

Implementing the Exercise

Start by making sure you are in your local clone of the Git repository, and in its exercises/03_html/3.3-method-based-routing directory where this instructions.html file is located. Then:

  • In this directory run the command npm init to initialize your Node project. Use the default values proposed by npm by pressing Enter.
  • Write your server code into index.js file in this directory.
  • In exercise 2.6 Routing your server responded differently based on the URL. This time you will choose your response based on the http method of the request:
    • Any GET request should be responded with the contents of the get.html file in this directory.
    • Any POST request should be responded with the contents of the post.html file in this directory.
    • Any other request should be responded with a status code 405 and the response should also have a header: Allow: GET, POST
  • Every GET and POST response should also have Content-Type: text/html and status code 200.

Remember to commit and push all changes to your Git repository before you submit your work in Plussa.

About Testing

Once your server is up and running you can test it with the index.html that you created in the previous exercise 3.2 HTML Forms. Open the file in your browser and submit the first form to test the response to the GET method and submit the second form to test POST response. Of course Postman or curl or some other similar software can also be used to test the server.
